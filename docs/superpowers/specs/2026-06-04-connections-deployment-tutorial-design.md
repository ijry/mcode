# Connections Deployment Tutorial Design

**Date:** 2026-06-04  
**Scope:** `mcode-app` 新增连接弹窗内增加全局可见的部署教程入口与说明弹窗  
**Primary Files:** `mcode-app/src/pages/connections/index.vue`

---

## Problem

当前 `mcode-app` 的“新增连接”弹窗只提供手动配置和扫码连接两个分段，没有告诉用户电脑端如何准备可连接的 `codeg` 服务。

这会导致两个实际问题：

- 首次使用者不知道需要先在电脑端安装 `codeg`
- 用户不知道应从哪里开启 `web服务`，也不知道要复制什么信息到 APP 中

因此，新增连接入口虽然已经存在，但第一次配置直连时缺少必要的部署引导。

## Goal

在“新增连接”弹窗内增加一个所有分段都可见的“部署教程”入口，并在点击后展示简洁、可执行的说明弹窗，帮助用户完成电脑端准备工作。

本次需要满足以下目标：

- 教程入口在“手动配置”和“扫码连接”两个分段下都可见
- 点击入口后弹出独立教程弹窗，不打乱现有连接表单布局
- 教程内明确说明下载地址、开启 `web服务` 的路径、需要复制的连接信息
- 教程补充外网访问时可使用 `ngrok` 等内网穿透工具

## Non-Goals

- 不修改现有连接保存、配对、切换、在线状态逻辑
- 不引入图片步骤、截图轮播或富文本渲染
- 不增加后端配置项或远端可编辑教程内容
- 不在本次改动中增加复制按钮、自动填充按钮或检测桌面端状态的能力

## Options Considered

### 1. 标题下方增加全局教程入口，点击打开独立说明弹窗（采用）

**做法**

- 在“新增连接”弹窗标题下方增加轻量教程入口
- 教程入口不受“手动配置 / 扫码连接”切换影响
- 点击后打开一个二级 `u-popup` 说明弹窗

**优点**

- 所有模式都能看到入口
- 主表单不被长文案挤压
- 教程内容后续容易扩展

**缺点**

- 用户需要额外点击一次才能看到说明

### 2. 直接在新增连接弹窗内常驻显示完整教程

**做法**

- 在表单上方或下方直接渲染完整教程文案

**优点**

- 信息最直接，不需要额外点击

**缺点**

- 会压缩表单可视区域
- 在移动端底部弹窗中容易造成内容拥挤和滚动负担

### 3. 在保存按钮上方放提示卡片，点击打开教程

**做法**

- 在表单底部按钮前插入提示卡片，再打开教程弹窗

**优点**

- 与提交流程更靠近

**缺点**

- 入口权重偏低
- 在扫码分段下也不如标题下方稳定

## Chosen Design

采用 **方案 1**：在“新增连接”弹窗标题下方增加一个全局可见的“部署教程”入口，点击后打开独立教程弹窗。

这样可以在不破坏现有表单结构的前提下，把首次配置所需的关键说明放到用户最容易发现的位置。

## Detailed Design

### 1. 入口位置与可见性

教程入口放置在 `新增连接` 标题行下方、分段切换控件上方。

该入口具备以下特征：

- 在主新增连接弹窗打开时始终可见
- 不随 `subsectionIndex` 切换隐藏
- 风格保持轻量，避免喧宾夺主

推荐表现形式为一行说明式入口，例如：

- 主文案：`部署教程`
- 辅助文案：`首次使用前请先完成电脑端配置`

### 2. 教程弹窗结构

点击教程入口后，打开独立的 `u-popup` 弹窗，标题为 `部署教程`。

弹窗内容为静态步骤说明，按顺序展示以下三段内容：

1. 在自己电脑下载安装 `codeg`
   - 提供 GitHub 发布页链接：`https://github.com/xintaofei/codeg/releases/tag/v0.14.11`
2. 打开 `codeg`，点击右上角齿轮图标，然后点击 `web服务` 开启
   - 说明需要复制这里的连接地址和访问 `token`
3. 如果需要外网访问，可使用 `ngrok` 等内网穿透工具

教程弹窗只负责说明，不承担填写表单、状态检测或复制能力。

### 3. 主弹窗与教程弹窗的关系

主“新增连接”弹窗在打开教程弹窗时保持当前状态，不自动关闭。

预期交互如下：

1. 用户打开“新增连接”
2. 用户点击“部署教程”
3. 教程弹窗覆盖显示
4. 用户关闭教程弹窗
5. 用户回到原本的新增连接内容，继续填写连接信息

这样可以避免用户看完教程后重新进入表单。

### 4. 外链打开行为

教程中的 GitHub 下载地址需要支持点击跳转。

实现预期：

- 点击下载链接后，调用当前 uni-app 环境可用的外链打开能力
- 优先满足 H5 和当前 APP 运行环境下能正常打开浏览器
- 如果跳转失败，给出轻量 `toast` 提示

本次不要求在弹窗内嵌浏览器，也不要求对链接做额外包装页。

### 5. 文案约束

本次教程文案直接固化在前端页面中，不通过配置中心、接口或本地多文件拆分加载。

固定文案包含以下关键信息：

- 下载 `codeg`
- 进入右上角齿轮设置
- 开启 `web服务`
- 复制连接地址和访问 `token`
- 外网访问可用 `ngrok`

其中用户原始描述中的 `nrgok` 按常见工具名修正为 `ngrok`。

## Data Flow

1. 用户点击“新增连接”
2. 页面显示现有新增连接弹窗
3. 用户点击“部署教程”入口
4. 页面将教程弹窗状态设为打开
5. 用户阅读步骤说明
6. 用户点击关闭或遮罩关闭教程弹窗
7. 页面返回新增连接主弹窗上下文
8. 用户继续填写连接信息或切换分段

## Files

- Modify: `mcode-app/src/pages/connections/index.vue`

## State Changes

页面新增一个本地 UI 状态即可，例如：

```ts
const showTutorialPopup = ref(false)
```

该状态仅用于控制教程弹窗显隐，不参与持久化，不影响连接数据模型。

## Error Handling

- 如果外链打开失败，显示 `toast` 提示用户手动复制或稍后重试
- 如果教程弹窗打开或关闭失败，不应影响主新增连接弹窗和连接表单使用
- 不对下载链接可达性做运行时校验，避免引入额外网络依赖

## Testing Strategy

手动验证以下场景：

1. 打开新增连接弹窗
   - 预期：教程入口在标题下方可见

2. 切换到“手动配置”
   - 预期：教程入口仍然可见

3. 切换到“扫码连接”
   - 预期：教程入口仍然可见

4. 点击教程入口
   - 预期：弹出标题为 `部署教程` 的说明弹窗

5. 关闭教程弹窗
   - 预期：返回新增连接主弹窗，已填写内容不丢失

6. 点击下载链接
   - 预期：触发外部链接打开

7. 链接打开失败场景
   - 预期：出现轻量错误提示，不影响主表单继续使用

## Acceptance Criteria

- “新增连接”弹窗内存在一个全局可见的“部署教程”入口
- 该入口在“手动配置”和“扫码连接”两个分段下都可见
- 点击入口会打开独立教程弹窗
- 教程弹窗包含下载地址、开启 `web服务`、复制连接地址和访问 `token`、`ngrok` 外网访问说明
- 关闭教程后，用户可以直接回到新增连接主弹窗继续操作
- 本次改动不影响现有连接保存与切换逻辑
