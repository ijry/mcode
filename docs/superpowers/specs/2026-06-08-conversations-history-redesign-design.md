# 会话页历史模式重构设计

## 背景

当前会话页首屏已经朝设计稿的轻列表方向收敛，但“历史会话”模式仍然使用 `up-cate-tab` 做左右双栏切换。这个结构更像管理台筛选页，不像移动端会话列表页，视觉上也和首屏样式割裂。

用户要求历史会话继续按项目分组展示，但不要使用 `up-index-list` 或 `up-cate-tab` 这类不匹配当前信息结构的承载方式。

## 目标

- 移除历史模式中的左右双栏 `up-cate-tab` 结构
- 保留“当前连接下，按项目分组查看历史会话”的能力
- 让历史模式的视觉语言与首屏会话卡统一
- 让历史模式更像移动端纵向列表页，而不是切页式后台界面

## 非目标

- 不改历史会话数据来源和加载逻辑
- 不改首屏分组逻辑
- 不引入新的跨页面状态管理
- 不重做会话详情页

## 现状问题

1. `up-cate-tab` 的左侧 tab 导航占据了大量横向空间，削弱了会话内容本身。
2. 历史模式的顶部栏、项目切换栏、卡片列表层次过多，视觉节奏过重。
3. 当前历史模式与首屏会话列表在卡片样式、留白、信息密度上不统一。
4. `up-index-list` 更适合字母或短锚点索引，不适合长项目名分组，因此不作为本次方案。

## 最终结构

历史模式改为单列滚动布局：

1. 顶部轻量返回栏
2. 历史会话滚动区
3. 项目分组标题
4. 项目分组下的会话卡片列表

不再出现左侧 tab、右侧 page pane 结构。

## 顶部返回栏

- 保留“返回分组”操作
- 中间显示当前连接名
- 右侧保留弱化的“新建”入口
- 返回栏做成一条轻量白卡，圆角小于首屏大卡，避免占据太多首屏

建议信息布局：

- 左侧：返回箭头 + `返回分组`
- 中间：当前连接名，单行截断
- 右侧：`新建`

## 项目分组列表

每个项目作为一个分组段落：

- 顶部是轻量项目标题
- 下方是该项目下的历史会话卡片
- 没有历史会话的项目不渲染

分组之间通过留白区分，而不是通过大容器或分隔面板区分。

## 历史会话卡片

历史会话卡片与首屏会话卡统一视觉基线：

- 白底
- 小圆角
- 轻阴影
- 紧凑纵向节奏

单张卡片结构：

- 左侧：会话图标块
- 中间主标题：会话标题
- 中间副标题：智能体类型 + 更新时间
- 右侧：轻量进入提示，优先是箭头；如果保留更多菜单，则菜单弱化

状态展示规则：

- 历史会话不强调“运行中”动态状态
- 如需保留状态，仅使用弱标签，不压过标题内容

## 视觉规则

- 整页继续使用首屏同款浅灰底
- 顶部返回栏为轻白卡，不做厚重阴影
- 项目标题为浅色辅助文字，不做块级底色
- 会话卡与首屏列表卡统一色彩、圆角和间距节奏
- 不引入蓝色大面积强调块

## 交互规则

- 点击历史会话卡，直接打开该会话
- 点击卡片右侧菜单，才触发删除、重命名等操作
- 点击返回栏左侧返回，回到首屏分组列表
- 返回首屏后尽量保留之前的搜索关键字与滚动上下文
- 点击右上“新建”，默认在当前连接上下文创建会话

## 数据组织

继续复用现有数据结构：

- 当前连接组：`selectedConnectionGroup`
- 项目集合：`projects`
- 项目下会话：`project.conversations`

展示层改造原则：

- 不再构建 `up-cate-tab` 专用 `tabList`
- 不再依赖 `currentTab`
- 直接遍历 `projects` 渲染项目分组和其会话列表

## 实现范围

本次改动范围限制在：

- `mcode-app/src/pages/conversations/index.vue`

如需清理仅服务于 `up-cate-tab` 的展示辅助逻辑，也限制在同文件中完成，不扩散到其他页面或服务层。

## 验收标准

1. 历史模式不再显示左右双栏 `up-cate-tab`
2. 历史会话按项目分组纵向展示
3. 历史会话卡与首屏会话卡风格统一
4. 顶部栏支持返回和新建
5. 点击历史会话可正常进入详情页
6. 删除、重命名等菜单交互继续可用
7. 构建通过，无新增运行时报错
