# Connections Tab iOS Redesign Design

**Date:** 2026-06-08  
**Scope:** `mcode-app` 连接 tab 的整页视觉重写  
**Primary Files:** `mcode-app/src/pages/connections/index.vue`
**Reference Design:** UI MCP project `projects/16101400267206960868`, screens `0d13f0b1c330459499b231f0289b7f2d` and `b1ea20c9cd1143b88e4976b80c3f9956`

---

## Problem

当前连接页功能完整，但视觉层级偏旧，信息密度和卡片样式都更像早期工具页：

- 顶部缺少明确的页面主标题层级
- 连接列表的卡片、状态、菜单按钮之间缺少一致的 iOS 风格秩序
- 空状态、新增入口、教程入口和列表之间的视觉关系不够统一
- 页面里已有的教程与表单能力没有形成一个完整的连接管理体验

## Goal

在不改变任何连接行为的前提下，把连接 tab 重写成更接近 UI MCP 参考稿的 iOS 风格连接管理页。

目标包括：

- 保留现有新增、编辑、删除、切换、在线状态逻辑
- 统一顶部、卡片、状态徽标、空状态、弹窗和操作菜单的视觉语言
- 让“新增连接”入口更醒目，但不改变现有交互流程
- 让页面在 H5 和移动端都保持干净、克制、可读

## Non-Goals

- 不改连接数据模型
- 不改连接保存、配对、切换、断开、删除逻辑
- 不改后台接口或网关能力
- 不新增远端配置
- 不改 tabBar 路由和底部导航行为
- 不把页面改成新的信息架构或多页流程

## Chosen Direction

采用参考稿里的 **iOS 风格浅色卡片布局**，但只保留与现有数据匹配的部分：

- 顶部品牌区 + 主按钮
- 连接列表卡片
- 添加新设备卡片
- 教程入口与教程弹窗
- 轻量信息卡作为页面补充

不直接照搬参考稿里与本地数据无关的内容，避免凭空引入新的数据源或功能负担。

## Detailed Design

### 1. 页面顶部

页面顶部改成轻量、分层明确的头部：

- 左侧显示品牌/产品标识
- 右侧显示主行动按钮 `新增连接`
- 页面主标题使用 `我的连接`
- 副标题使用简短说明，强调“管理与切换”

视觉上采用更大的留白、更明显的标题权重和更柔和的背景过渡，避免旧式工具页的压迫感。

### 2. 连接列表

每个连接项改成更接近 iOS 卡片的样式：

- 卡片圆角更大
- 使用浅阴影或浅色分层，而不是重边框
- 左侧图标容器更明确
- 名称、状态、地址分成清晰的三层
- 右侧保留更多菜单，但按钮更轻

卡片内信息层级：

1. 连接名称
2. 在线状态徽标
3. 模式 + 地址摘要
4. 次级入口 `管理连接`

状态显示保留现有在线逻辑，但视觉上改成胶囊/徽标风格，而不是简单圆点。

### 3. 新增入口与空状态

在列表中保留一个显眼的 `添加新设备` 卡片，作为新增连接的主入口之一。

空状态时：

- 保留空状态提示
- 让新增按钮成为唯一主要操作
- 维持与列表态一致的圆角、颜色和按钮语言

### 4. 教程入口与教程弹窗

继续保留“部署教程”入口，但把它和整体卡片风格统一起来：

- 入口作为独立的浅色提示卡
- 标题和说明更简短
- 作为新用户的辅助引导，不占据主表单空间

教程弹窗保持现有内容，但视觉统一到新的浅色卡片语言。

### 5. 辅助信息卡

参考稿中的信息卡可以保留为轻量补充，但只允许使用已有本地资源或静态文案：

- 可复用现有插图资源
- 可显示一条简短的部署/状态提示
- 不新增远程数据请求

如果页面空间不足，辅助信息卡应优先被压缩，而不是影响连接列表和新增入口。

### 6. 弹窗与操作菜单

保留现有弹窗行为，但统一视觉：

- 新增/编辑连接弹窗继续使用底部弹出层
- 操作菜单继续使用 action sheet
- 教程弹窗继续使用居中弹窗

三者都应与页面主卡片保持同一套圆角、间距和浅色分层规则。

## Files

- Modify: `mcode-app/src/pages/connections/index.vue`

## State

本次重写不引入新的业务状态。

继续复用现有状态：

- `showAddPopup`
- `showTutorialPopup`
- `subsectionIndex`
- `loading`
- `showActionSheet`
- `currentConnectionIndex`
- `editingConnectionKey`
- `connectedMap`
- `onlineMap`

## Error Handling

- 连接失败时继续沿用现有 toast，不改变错误语义
- 教程弹窗打开失败不应阻塞主连接弹窗
- 菜单按钮、切换和删除失败时，页面样式不应回退或闪烁

## Testing Strategy

手动验证以下场景：

1. 打开连接 tab
   - 预期：顶部标题、主按钮、卡片层级与参考稿一致

2. 有连接数据时
   - 预期：列表卡片、在线徽标、更多菜单正常显示

3. 无连接数据时
   - 预期：空状态和新增入口清晰可用

4. 点击 `新增连接`
   - 预期：底部弹窗正常打开，布局与页面风格一致

5. 点击 `部署教程`
   - 预期：教程弹窗正常打开，关闭后返回原表单上下文

6. 对连接执行连接、断开、编辑、删除
   - 预期：功能与现状一致，仅视觉变化

7. 切换 H5 与移动端预览
   - 预期：无横向滚动、无卡片溢出、文本不截断错位

## Acceptance Criteria

- 连接 tab 完成 iOS 风格视觉重写
- 现有连接功能保持不变
- 顶部、列表、空状态、教程入口和弹窗统一为同一套视觉语言
- 页面在 H5 和移动端都可正常使用
- 没有引入新的业务数据源或连接逻辑
