# 连接页项目列表与项目会话导航

## 背景

连接首页卡片底部原先展示 `管理连接`，但当前产品没有对应的独立连接管理落地页，用户从连接卡片出发时更实际的目标是查看该连接下的项目与项目会话。

本次改动把连接卡片次级入口改成 `项目列表`，并串起：

1. 连接首页 -> 项目列表页
2. 项目列表页 -> 项目会话列表页
3. 项目会话列表页 -> 既有会话详情页

## 数据流与协议

- 页面之间不再只传 `mode::url` 的简化连接 key，而是显式传递完整连接上下文：
  - `name`
  - `mode`
  - `url`
  - `directToken`
  - `pairCode / pairSecret`
  - `relaySession`
- 新增 `connectionContext` 工具，负责：
  - 编码/解码页面路由里的连接上下文；
  - 按连接模式恢复 gateway；
  - 为直连或中继连接生成稳定 `instanceKey`；
  - 在中继 session 刷新后回写本地 `mcode_connections`。
- 项目列表页远端调用 `list_open_folder_details` 获取当前连接下项目。
- 项目会话列表页远端调用 `list_all_conversations`，只请求单个 `folderId`。
- 项目统计复用本地 SQLite `conversations` 摘要表，不新增服务端接口。

## UI 行为

- 连接卡片底部文案从 `管理连接` 改为 `项目列表`。
- 点击连接卡片时：
  - 若该连接尚未建立本地 linked 状态，先执行现有连接动作；
  - 然后进入该连接的项目列表页。
- 项目列表页每个项目展示：
  - 该项目会话总数；
  - 正在进行中的会话数。
- 点击项目后进入该项目的会话列表页。
- 项目会话列表页点击单条会话后，进入既有会话详情页。

## 统计口径

- `会话数量`：本地摘要表里，该 `folderId` 下未删除摘要总数。
- `正在进行的会话数量`：
  - `in_progress`
  - `pending_review`
- 这里的 `pending_review` 对应“等待用户输入/审批但尚未结束”的口径。

## 兼容性

- 会话详情页保留原有 `connectionKey = mode::url` 路由兼容。
- 若详情页收到的是新的完整连接上下文字符串，会优先按完整上下文恢复认证与远端描述。
- 旧入口不需要同步改造即可继续工作。

## iOS / Android 原生复刻要求

- 原生页跳转时，不要只传连接列表索引，也不要依赖“全局当前连接”。
- 应显式随路由或页面参数传递完整连接上下文，至少包含：
  - 连接模式；
  - base URL；
  - 认证所需 token / relay session；
  - 连接显示名。
- 项目统计优先复用本地会话摘要缓存；本地无缓存时允许显示 `0 / 0`，待同步后刷新。
- 项目会话页进入详情页时，继续把连接上下文传下去，保证详情页恢复实时连接、上传附件和中继刷新时不丢上下文。
